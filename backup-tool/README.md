# 실시간 폴더 백업

지정한 디렉토리를 실시간으로 감시하여 다른 경로로 백업하는 Python GUI 프로그램입니다.
선택 사항으로 1시간 주기로 구글 드라이브에 변경분을 미러링할 수 있습니다.

## 기능

1. 백업 대상 디렉토리 지정
2. 백업 저장 디렉토리 지정
3. 제외할 파일 / 하위 디렉토리 패턴 지정 (예: `*.tmp`, `node_modules`)
4. 실시간 파일 변경 감지 후 자동 백업
5. (선택) 구글 드라이브 1시간 주기 자동 백업 — 변경된 파일만 업로드

백업 방식은 **누적(안전 모드)** 입니다. 원본에서 파일이 삭제·이동되어도 백업본은 보존됩니다.
추가·수정된 내용만 백업에 반영됩니다.

백업은 저장 디렉토리 아래에 **날짜별 하위 폴더(`YYYYMMDD`)** 를 만들어 그 안에 저장합니다.
초기 동기화는 시작한 날짜 폴더에, 실시간 감시는 변경이 발생한 그날의 날짜 폴더에 백업됩니다.
예: `D:\backup\20260515\문서\보고서.docx`

## 설치

```
pip install -r requirements.txt
```

## 실행

```
python main.py
```

## 사용법

1. **백업 대상 디렉토리** 와 **백업 저장 디렉토리** 를 지정합니다.
   - 두 경로는 서로 같거나 포함 관계일 수 없습니다(무한 복사 방지).
2. 필요하면 **제외 패턴** 을 추가합니다. 대상 디렉토리 기준 상대경로에 대한 glob 패턴이며,
   디렉토리 이름만 적으면 그 하위 전체가 제외됩니다.
   - 사용자 지정 패턴과 별개로, Office 가 저장 중 만드는 임시·잠금 파일은 **항상 자동 제외**
     됩니다: `~$...` (잠금 파일), `*.tmp` (임시 파일), 8자리 hex 이름 파일(예: `ACA7B500`).
3. 필요하면 **구글 드라이브 연동** 토글을 켭니다(설정 영역 상단). 자세한 설정은 아래
   "구글 드라이브 연동" 절을 참고하세요.
4. **감시 시작** 을 누르면 먼저 전체 초기 동기화를 수행한 뒤 실시간 감시로 들어갑니다.
   - 구글 드라이브 토글이 켜져 있으면 감시 시작 직후 1회 + 이후 1시간마다 변경분을 업로드합니다.
5. 설정(경로·제외 목록)은 `config.json` 에 자동 저장되어 다음 실행 시 복원됩니다.
   구글 드라이브 로그인 상태는 `credentials/token.json` 의 존재로 유지됩니다.
6. 프로젝트 폴더의 `log/` 안에 **하루 단위** 로그 파일 `backup_YYYYMMDD.log` 가 생성되며,
   화면 로그가 갱신될 때마다 같은 내용이 파일에도 기록됩니다. 같은 날에 감시를 여러 번
   재시작해도 모든 로그가 한 파일에 누적됩니다(세션 사이에 `=== 감시 시작/종료 ===` 마커가
   삽입되어 구분할 수 있습니다).

오류가 발생하면 로그 창에 빨간색으로 표시되고, 진행을 막는 오류는 메시지 상자로 안내됩니다.
감시 / 초기 동기화 중에는 경로 입력칸·찾아보기 버튼·제외 목록·추가/삭제 버튼·구글 드라이브
토글이 모두 잠깁니다(중지하면 다시 활성화).

## 구글 드라이브 연동

개인 Gmail 계정의 OAuth 데스크톱 앱 방식으로 인증합니다. 서비스 계정은 저장 용량 할당량이
없어 사용하지 않습니다.

### 최초 설정 (1회만)

1. [Google Cloud Console](https://console.cloud.google.com/) 에서 프로젝트 생성/선택
2. **API 및 서비스 → 라이브러리** → "Google Drive API" 검색 → 사용 설정
3. **API 및 서비스 → OAuth 동의 화면**
   - User Type: **External**
   - 앱 이름·연락처 입력
   - **테스트 사용자** 에 본인 Gmail 추가 (이게 없으면 동의 화면에서 거부됨)
4. **API 및 서비스 → 사용자 인증 정보 → + 만들기 → OAuth 클라이언트 ID**
   - 애플리케이션 유형: **데스크톱 앱**
   - 만든 뒤 JSON 다운로드
5. 다운로드한 JSON 을 `credentials/oauth_client.json` 으로 저장
6. 업로드 대상이 될 Drive 폴더를 만들고 본인 Gmail 에 편집자 권한이 있는지 확인
   (본인 소유 폴더라면 자연히 OK). 폴더 ID 는 `app/gdrive.py` 의 `FOLDER_ID` 상수로 지정합니다.

### 사용

- 토글을 켜면 브라우저가 열려 Google 로그인·동의 화면이 표시됩니다. 동의 후
  `credentials/token.json` 이 생성되고 이후 실행에서는 자동 재사용·갱신됩니다.
- 로그인된 계정의 이메일이 토글 옆 라벨에 표시됩니다.
- 토글을 끄면 토큰을 폐기 시도(`oauth2.googleapis.com/revoke`)한 뒤
  `credentials/token.json` 을 삭제합니다.
- 프로그램을 다시 실행하면 `token.json` 이 있을 경우 토글이 자동으로 켜진 상태로 복원됩니다.

### 업로드 동작

- 감시 시작 직후 1회 + 이후 1시간 간격으로 업로드합니다.
- **변경된 파일만** 업로드됩니다. `watchdog` 이 감지한 변경 파일과 초기 동기화로 복사된 파일이
  내부 큐에 쌓이고, 매 사이클마다 큐 스냅샷을 비워 그 파일들만 올립니다.
- 같은 이름의 Drive 폴더/파일은 재사용·업데이트(덮어쓰기)되어 중복 항목이 쌓이지 않습니다.
- 사이클 도중 발생한 새 변경은 자연스럽게 다음 사이클로 미뤄집니다.
- 일시적 실패(네트워크 등)는 큐에 되돌아가 다음 1시간 후에 자동 재시도됩니다.
- 큐가 비어 있는 사이클에는 "올릴 변경 파일이 없어 이번 주기는 건너뜁니다" 로그를 남기고
  타이머는 그대로 다음 주기를 기다립니다.
- 감시를 중지하면 1시간 타이머도 함께 정지합니다(로그인 상태와 토큰은 유지).

## 시스템 트레이

- 창을 닫으면 **"트레이로 보내기 / 종료 / 취소"** 를 묻습니다. 트레이로 보내면 백그라운드에서
  감시를 계속하고, 트레이 아이콘을 클릭하거나 우클릭 메뉴의 "열기" 로 창을 다시 띄울 수 있습니다.
- 트레이 우클릭 메뉴에서 감시 시작/중지와 완전 종료가 가능합니다.
- 백업·오류 발생 시 트레이 풍선 알림이 표시됩니다. 백업 알림은 **창이 트레이에 숨겨져 있을 때만**
  뜨고, 너무 잦지 않도록 일정 간격으로 묶어서 표시됩니다. 오류 알림은 항상 표시됩니다.
- 시스템 트레이를 지원하지 않는 환경에서는 창을 닫으면 그대로 종료됩니다.

## 프로젝트 구조

```
backup-tool/
├── main.py              # 엔트리포인트
├── requirements.txt
├── config.json          # 실행 중 자동 생성되는 설정 파일
├── credentials/
│   ├── oauth_client.json  # 사용자가 직접 배치 (Cloud Console 에서 발급)
│   └── token.json         # 최초 로그인 후 자동 생성·갱신
├── log/                 # 감시 시작 시 자동 생성되는 세션 로그 폴더
└── app/
    ├── errors.py        # 앱 전용 예외 클래스
    ├── config.py        # 설정 모델 + JSON 로드/저장 + 검증
    ├── backup_engine.py # 복사 로직, 제외 필터, 초기 동기화
    ├── watcher.py       # watchdog 기반 실시간 감시
    ├── gdrive.py        # 구글 드라이브 OAuth + 업로드
    ├── logger.py        # 감시 세션별 로그 파일 기록
    └── ui/
        ├── signals.py     # 스레드 간 신호 브리지
        └── main_window.py # 메인 윈도우
```
